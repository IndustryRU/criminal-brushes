# Criminal Brushes Docs

Единая папка документации проекта `Criminal Brushes`: продукт, архитектура, команда, релизы и публикация.

## Структура

- [01-product-brief.md](01-product-brief.md) - продуктовая рамка, аудитория, позиционирование и MVP.
- [02-technical-spec.md](02-technical-spec.md) - базовое техническое задание для разработки.
- [03-architecture.md](03-architecture.md) - архитектура Flutter + Supabase и структура репозитория.
- [04-database-schema.md](04-database-schema.md) - минимальная модель данных Supabase/Postgres.
- [05-design-system.md](05-design-system.md) - дизайн-направление, UI-токены и компоненты.
- [06-team-workflow.md](06-team-workflow.md) - командный процесс, роли, Git flow и правила разработки.
- [07-environment-setup.md](07-environment-setup.md) - настройка локальной и общей среды.
- [08-release-and-store-checklist.md](08-release-and-store-checklist.md) - чеклист релиза для Web, RuStore, Google Play и App Store.
- [09-roadmap.md](09-roadmap.md) - поэтапный план работ от подготовки до запуска.
- [10-implementation-start.md](10-implementation-start.md) - первый setup разработки и аудит текущей машины.
- [11-new-developer-onboarding.md](11-new-developer-onboarding.md) - подключение нового разработчика с пустой машины.
- [12-git-sync-playbook.md](12-git-sync-playbook.md) - синхронизация проекта через Git и промпты для Codex.
- [13-supabase-setup.md](13-supabase-setup.md) - создание Supabase-проекта и применение начальной схемы БД.
- [14-fvm-version-lock.md](14-fvm-version-lock.md) - фиксация Flutter-версии через FVM для всей команды.
- [15-cloud-supabase-workflow.md](15-cloud-supabase-workflow.md) - работа с облачным Supabase без Docker через API.
- [phases/](phases/README.md) - подробные технические задания и критерии приемки отдельных фаз.

## Как поддерживать документы

- Все продуктовые и технические решения фиксируем здесь до или сразу после реализации.
- Если решение влияет на архитектуру, базу данных, релиз или командный процесс, обновляем соответствующий файл.
- Секреты, токены, ключи API и персональные данные в документацию не добавляем.
- Даты и требования маркетплейсов перепроверяем перед публикацией, потому что правила магазинов меняются.
